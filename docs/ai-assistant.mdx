Athas includes a built-in AI assistant for coding help through Harness.

## Features

- **Chat Interface**: Conversational AI for coding questions
- **Multiple Providers**: OpenAI, Anthropic, OpenRouter, Gemini, Grok, Ollama, and more
- **Context Awareness**: AI can use files you reference
- **Pi Native Restore**: Pi sessions reopen with their shared session identity plus restored model and thinking settings

## Getting Started

<Steps>
  <Step>
    Open Harness from the footer button or `Cmd+R` / `Ctrl+R`
  </Step>
  <Step>
    Configure your AI provider in **Settings > AI**
  </Step>
  <Step>
    Start chatting or asking questions about your code in Harness
  </Step>
</Steps>

## Configuration

Configure AI settings in **Settings > AI**:

- **Provider**: Select your preferred AI service
- **Model**: Choose the specific model
- **Temperature**: Control response creativity (0-2)
- **Max Tokens**: Limit response length

## Pi Native Sessions

When you use Pi in Harness with the native runtime, Athas reads the same session files used by standalone Pi. Reopened sessions restore their conversation history, active model, and thinking level from the shared Pi session data instead of starting from an empty Athas-only chat state.

Forking a Pi-native conversation keeps the restored model and thinking defaults, but starts a new native branch instead of continuing the same underlying Pi session file.

When you switch between restored Pi-native chats in the same Harness scope, Athas now rebinds the native route to the selected shared Pi session instead of accidentally continuing whichever Pi session was already live on that route.
