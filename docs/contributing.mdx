## Development Setup

### Common Prerequisites

- Git
- Rust (via rustup)
- Node.js 18+ and bun

### Platform Setup

<Tabs items={["macOS", "Linux", "Windows"]}>
  <Tab value="macOS">
    **Prerequisites**

    - Xcode Command Line Tools
    - Rust (via rustup)
    - Node.js 18+ and bun

    **Install dependencies**

    ```bash
    # Install Xcode CLI tools
    xcode-select --install

    # Install Rust
    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
    ```

    ```bash
    brew install pkg-config
    ```


  <Tab value="Windows">
    **Prerequisites**

    - Visual Studio Build Tools
    - Rust (via rustup)
    - Node.js 18+ and bun
    - WebView2

    **Install dependencies**

    1. Install [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) with "Desktop development with C++".
    2. Install Rust from [rustup.rs](https://rustup.rs/).
    3. Install bun:
       ```powershell
       powershell -c "irm bun.sh/install.ps1 | iex"
       ```
    4. Ensure WebView2 is installed (usually preinstalled on Windows 10/11).

### Clone and Run

```bash
git clone https://github.com/athasdev/athas.git
cd athas
bun install
bun dev
```

### Linux Virtual Desktop UI Testing

If you're working on UI bugs in a headless Linux environment or VPS, Athas includes a repeatable virtual desktop launcher.

Recommended extra packages:

- `xvfb`
- `openbox`
- `xdotool`
- `wmctrl`
- `scrot`

Start the session:

```bash
bun run athas-ui-test:check
bun run athas-ui-test
```

Useful follow-up commands:

```bash
bun run athas-ui-test:status
bun run athas-ui-test:screenshot -- /tmp/athas-ui.png
bun run athas-ui-test:logs
bun run athas-ui-test:stop
```

The launcher starts:

- `Xvfb` on `:99` by default
- `openbox`
- `bun dev:app`

You can override the defaults with environment variables:

```bash
ATHAS_UI_DISPLAY=:100 ATHAS_UI_SCREEN=1920x1080x24 bun run athas-ui-test
ATHAS_UI_APP_CMD="bun tauri dev --no-watch" bun run athas-ui-test
```

If the preflight reports missing Tauri system libraries, install the Linux dependencies first:

```bash
bash scripts/linux/setup.sh
```

Once the session is running, desktop automation tools can target the virtual display directly:

```bash
DISPLAY=:99 XAUTHORITY=/tmp/athas-ui-test-$USER/Xauthority xdotool search --name Athas
DISPLAY=:99 XAUTHORITY=/tmp/athas-ui-test-$USER/Xauthority wmctrl -lp
```

## Release Process

1. Update version in `package.json` and `src-tauri/Cargo.toml`.
2. Create a git tag: `git tag v1.0.0`.
3. Push the tag: `git push origin v1.0.0`.

### GitHub Actions

The release workflow automatically:

- Builds for macOS, Linux, and Windows
- Creates GitHub release
- Uploads installers and binaries

### Versioning

We use semantic versioning: `MAJOR.MINOR.PATCH`.

- **MAJOR**: Breaking changes
- **MINOR**: New features (backwards compatible)
- **PATCH**: Bug fixes
